## 정규표현식 테스터를 비교하는 이유

정규표현식(regex)은 텍스트 처리, 로그 분석, 데이터 검증에 필수적인 도구입니다. regex101은 가장 널리 알려진 온라인 정규식 테스터이지만, 모든 상황에 최선은 아닙니다.

beomanro의 [정규표현식 테스터](/tools/net/regex-tester/)는 가볍고 프라이버시를 중시하는 대안입니다.

## 상세 비교

### 프라이버시

가장 큰 차이점입니다. regex101은 입력한 **모든 텍스트와 패턴을 서버로 전송**합니다. 로그 데이터, 개인정보가 포함된 텍스트를 테스트할 때 데이터 유출 위험이 있습니다.

beomanro는 모든 처리를 브라우저에서 수행하므로 **어떤 데이터도 외부로 나가지 않습니다**.

### 속도와 가벼움

beomanro는 단일 페이지로 즉시 로딩됩니다. regex101은 다양한 기능을 제공하는 대신 초기 로딩과 UI 반응이 상대적으로 느립니다.

### 한글 지원

beomanro는 UI와 플래그 설명을 한국어로 제공합니다. 정규표현식을 처음 배우는 사용자에게 유용합니다.

### regex101의 강점

공정하게 비교하면, regex101은 **디버거**, **패턴 라이브러리**, **다중 엔진 지원**(PCRE, Python, Go 등), **URL을 통한 패턴 공유** 같은 고급 기능을 제공합니다. 복잡한 정규식 개발에는 regex101이 더 강력합니다.

## 언제 어떤 도구를 쓸까?

### beomanro가 유리한 경우

- 민감한 데이터(로그, 개인정보)를 테스트할 때
- 빠르게 간단한 패턴을 확인할 때
- 한글 환경이 필요할 때
- 오프라인/폐쇄망 환경에서 작업할 때

### regex101이 유리한 경우

- 복잡한 정규식을 단계별 디버깅할 때
- Python, PCRE 등 특정 엔진의 동작을 확인할 때
- 패턴을 팀원과 URL로 공유할 때

## 정규식 엔진마다 다르게 동작한다

같은 패턴도 엔진에 따라 결과가 다릅니다. 이는 도구 선택과도 직결됩니다.

- **룩비하인드** `(?<=...)`: JavaScript는 ES2018부터 지원합니다. 구형 환경·일부 엔진은 미지원.
- **유니코드 `\d`**: PCRE/Python의 유니코드 모드에서는 `\d`가 아라비아 숫자 외 문자도 포함할 수 있습니다.
- **재귀 패턴** `(?R)`: PCRE는 지원하지만 JavaScript는 지원하지 않습니다.

beomanro 테스터는 브라우저의 JavaScript 엔진으로 동작하므로 **실제 JS에서 쓸 정규식**을 그대로 검증하는 데 정확합니다. 반대로 Python·PCRE 동작을 확인해야 한다면 regex101의 다중 엔진이 맞습니다.

## 결론

regex101은 정규표현식의 "스위스 아미 나이프"입니다. 하지만 **프라이버시가 중요한 상황**, **빠른 확인**, **한글 환경**에서는 beomanro의 [정규표현식 테스터](/tools/net/regex-tester/)가 더 적합합니다. 두 도구를 상황에 맞게 병행하는 것이 가장 효율적입니다.

리눅스 관련 작업을 할 때는 [Cron 파서](/tools/net/cron-parser/)도 함께 활용해 보세요.
