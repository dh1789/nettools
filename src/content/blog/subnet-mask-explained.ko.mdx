> **이 글의 범위** — 서브넷 마스크의 비트 연산 원리, 클래스 체계, /31·/32 특수 프리픽스를 다룹니다. CIDR·VLSM 설계는 [CIDR와 VLSM 완벽 가이드](/blog/subnet-guide/), 실무 계산 절차는 [서브넷 계산 완전 가이드](/blog/subnet-calculation-guide/)를 참고하세요.

## 서브넷 마스크의 역할

네트워크에서 두 장비가 통신할 때, 운영체제는 먼저 "상대방이 같은 네트워크에 있는가?"를 판단합니다. 이 판단에 사용되는 것이 바로 **서브넷 마스크**입니다.

서브넷 마스크는 IP 주소를 두 부분으로 나눕니다:

- **네트워크 부분**: 같은 네트워크에 속한 장비들이 공유하는 주소
- **호스트 부분**: 네트워크 내에서 각 장비를 구분하는 주소

## 비트 수준에서 이해하기

서브넷 마스크 `255.255.255.0`을 이진수로 변환하면:

```
11111111.11111111.11111111.00000000
```

연속된 `1`이 네트워크 부분, 연속된 `0`이 호스트 부분입니다. IP 주소와 서브넷 마스크를 **AND 연산**하면 네트워크 주소를 구할 수 있습니다:

```
IP 주소:     192.168.1.100  = 11000000.10101000.00000001.01100100
서브넷 마스크: 255.255.255.0  = 11111111.11111111.11111111.00000000
──────────────────────────────────────────────────────────────────
네트워크 주소: 192.168.1.0    = 11000000.10101000.00000001.00000000
```

두 장비의 네트워크 주소가 동일하면 같은 서브넷에 있는 것입니다.

## IP 주소 클래스

초기 인터넷은 클래스(Class) 기반으로 IP를 할당했습니다:

| 클래스 | 첫 옥텟 범위 | 기본 마스크 | 네트워크 수 | 호스트 수 |
|--------|-------------|-----------|-----------|---------|
| A      | 1~126       | /8        | 126       | 16,777,214 |
| B      | 128~191     | /16       | 16,384    | 65,534 |
| C      | 192~223     | /24       | 2,097,152 | 254 |

하지만 클래스 기반 할당은 IP 낭비가 심해서, 현재는 **CIDR**(Classless Inter-Domain Routing)이 표준입니다.

## 서브네팅이 필요한 이유

하나의 /24 네트워크(254개 호스트)에 모든 장비를 넣으면:

1. **브로드캐스트 폭풍**: 모든 장비가 모든 브로드캐스트 패킷을 수신
2. **보안 취약**: 같은 네트워크 내에서 스니핑이 용이
3. **관리 어려움**: 문제 발생 시 전체 네트워크에 영향

서브네팅으로 네트워크를 분할하면 이 문제를 해결할 수 있습니다.

## 자주 사용하는 서브넷 마스크

| CIDR | 마스크 | 호스트 수 | 용도 |
|------|--------|---------|------|
| /30  | 255.255.255.252 | 2 | 포인트-투-포인트 링크 |
| /27  | 255.255.255.224 | 30 | 소규모 부서 |
| /24  | 255.255.255.0 | 254 | 일반 LAN |
| /16  | 255.255.0.0 | 65,534 | 대규모 네트워크 |

## 호스트 수는 왜 2개를 빼는가

프리픽스가 `/n`이면 호스트 비트는 `32 - n`개, 전체 주소는 `2^(32-n)`개입니다. 여기서 **2개를 빼서** 사용 가능 호스트 수를 구합니다.

```
사용 가능 호스트 = 2^(32-prefix) - 2
/24 → 2^8 - 2 = 254
/27 → 2^5 - 2 = 30
```

빼는 2개는 **네트워크 주소**(호스트 비트가 전부 0)와 **브로드캐스트 주소**(전부 1)입니다. 둘 다 개별 장비에 배정할 수 없기 때문입니다.

예외도 있습니다.

- **/31**: 포인트-투-포인트 링크 전용. RFC 3021에 따라 네트워크·브로드캐스트 개념 없이 2개 주소를 양쪽 끝에 그대로 씁니다.
- **/32**: 단일 호스트(루프백, 로드밸런서 VIP 등). 주소 1개.
