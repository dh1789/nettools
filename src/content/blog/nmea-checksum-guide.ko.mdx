## NMEA 0183이란

**NMEA 0183**은 GPS 수신기, 선박 항법 장비, 어군 탐지기 같은 해양·측위 장비가 데이터를 주고받는 ASCII 직렬 통신 표준입니다. 한 줄이 하나의 "문장(sentence)"이고, 사람이 읽을 수 있는 텍스트로 되어 있습니다.

```
$GPGGA,092750.000,5321.6802,N,00630.3372,W,1,8,1.03,61.7,M,55.2,M,,*76
```

구조를 뜯어보면 이렇습니다.

| 부분 | 값 | 의미 |
|------|-----|------|
| 시작 구분자 | `$` | 표준 문장 시작 (AIS는 `!`) |
| 토커 + 문장 ID | `GPGGA` | GP=GPS, GGA=위치/고도 |
| 데이터 필드 | `092750.000,...` | 쉼표로 구분된 값들 |
| 체크섬 구분자 | `*` | 본문 끝 표시 |
| 체크섬 | `76` | 2자리 16진수 |

마지막 `*76`이 바로 이 글의 주제, **체크섬**입니다.

## 체크섬은 무엇을 위해 있는가

직렬 통신은 노이즈에 약합니다. 케이블 간섭, 잘못된 보율(baud rate), 버퍼 오버런으로 문자 하나가 깨지면 위치값이 통째로 틀어질 수 있습니다. 체크섬은 **전송 중 데이터가 변형됐는지 가볍게 검출**하기 위한 1바이트 무결성 값입니다.

수신기는 받은 본문으로 체크섬을 다시 계산해, 문장 끝의 `*XX`와 비교합니다. 다르면 그 줄을 버립니다.

> 주의: 체크섬은 "암호"가 아니고 "오류 정정"도 아닙니다. 단지 **깨졌는지 여부**만 알려주는 가벼운 검사입니다.

## 계산 원리 — 시작 구분자와 `*` 사이를 XOR

규칙은 의외로 단순합니다.

1. **시작 구분자(`$` 또는 `!`)와 `*` 사이**의 모든 문자를 고릅니다. 구분자와 `*` 자체는 **포함하지 않습니다.**
2. 각 문자의 ASCII 코드를 **순서대로 XOR(배타적 논리합)** 합니다.
3. 결과(0~255)를 **2자리 대문자 16진수**로 표기합니다.

### 손으로 따라 해보기

본문이 `AB` 두 글자라고 합시다.

- `A` = `0x41` = `0100 0001`
- `B` = `0x42` = `0100 0010`
- XOR = `0000 0011` = `0x03`

→ 체크섬은 `03`. XOR은 같은 비트끼리 0, 다른 비트끼리 1이 되는 연산이라, 문자가 하나라도 바뀌면 결과가 달라집니다. 그래서 변형 검출에 쓰입니다.

16진수 ↔ 2진수 변환이 헷갈리면 [진법 변환기](/tools/net/number-base-converter/)로 직접 확인해 보세요.

### 실제 문장으로 검증

위의 `$GPGGA,...` 문장이라면 `$`와 `*` 사이의 `GPGGA,092750.000,...,M,,` 전체를 XOR하면 정확히 `0x76` → `76`이 나옵니다. 직접 한 글자씩 누적해 보려면 [NMEA 체크섬 계산기](/tools/net/nmea-checksum/)의 "XOR 과정 표시"를 켜면 단계별 누적값을 볼 수 있습니다.

## 검증 모드 vs 계산 모드

NMEA 체크섬 도구는 입력에 `*XX`가 있느냐로 동작이 갈립니다.

- **검증(Validate)**: `$GPRMC,...*45`처럼 체크섬이 붙어 있으면, 계산값과 입력값을 비교해 **✅ 일치 / ❌ 불일치**를 표시합니다. 깨진 로그를 디버깅할 때 씁니다.
- **계산(Calculate)**: `GPGGA,123519,...`처럼 본문만 주면, 올바른 체크섬을 계산해 **완성 문장(`$...*47`)**을 만들어 줍니다. 직접 문장을 작성할 때 씁니다.

대소문자는 구분하지 않습니다. 표준은 대문자(`*4A`)를 권장하지만 `*4a`도 동일하게 처리됩니다.

## AIS(`!`) 문장도 같은 규칙

선박 자동 식별 장치(AIS)의 AIVDM/AIVDO 문장은 `$` 대신 `!`로 시작합니다.

```
!AIVDM,1,1,,B,177KQJ5000G?tO`K>RA1wUbN0TKH,0*5C
```

페이로드가 6비트 ASCII로 인코딩돼 있어 낯설어 보이지만, **체크섬 규칙은 표준 NMEA와 완전히 동일**합니다. `!`와 `*` 사이의 모든 문자(페이로드 내부의 백틱·꺾쇠 같은 특수문자 포함)를 XOR하면 `5C`가 나옵니다.

## 흔한 실수

- **`$`나 `*`를 XOR에 포함** — 가장 흔한 버그. 구분자는 제외해야 합니다.
- **한 자리 16진수** — `0x3` 같은 값을 `3`으로 쓰면 안 되고 `03`처럼 2자리로 채워야 합니다.
- **CR/LF를 본문에 포함** — 줄 끝 `\r\n`은 `*` 뒤에 오므로 XOR 대상이 아닙니다. 본문에 섞이지 않게 트림하세요.
- **체크섬 일치 = 데이터 정확, 이라는 착각** — XOR는 8비트뿐이라 충돌 가능성이 있고, 짝수 개 비트가 뒤집히면 못 잡습니다. 무결성의 가벼운 보증일 뿐입니다.

## 정리

NMEA 체크섬은 "시작 구분자와 `*` 사이를 XOR해서 2자리 대문자 16진수로" — 이 한 문장이 전부입니다. GPS든 AIS든 규칙은 같습니다. 로그를 디버깅하거나 직접 문장을 만들 때는 [NMEA 0183 체크섬 계산기/검증기](/tools/net/nmea-checksum/)로 여러 줄을 한 번에 검증·계산할 수 있습니다. 모든 계산은 브라우저 안에서만 이뤄지므로 입력값이 외부로 전송되지 않습니다.
