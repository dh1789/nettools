## 크론 표현식이란?

크론(Cron)은 유닉스/리눅스 시스템에서 작업을 예약 실행하는 스케줄러입니다. 크론 표현식은 **"언제 실행할지"**를 5개 필드로 정의하는 문법입니다.

```
┌───────── 분 (0-59)
│ ┌─────── 시 (0-23)
│ │ ┌───── 일 (1-31)
│ │ │ ┌─── 월 (1-12)
│ │ │ │ ┌─ 요일 (0-7, 0과 7은 일요일)
│ │ │ │ │
* * * * *
```

## 특수 문자 이해하기

| 문자 | 의미 | 예시 |
|------|------|------|
| `*`  | 모든 값 | `* * * * *` = 매분 |
| `,`  | 값 나열 | `1,15 * * * *` = 매시 1분, 15분 |
| `-`  | 범위 | `1-5 * * * *` = 1~5분 |
| `/`  | 간격 | `*/10 * * * *` = 10분마다 |

## 실무에서 자주 쓰는 패턴

### 매일 자정에 실행

```
0 0 * * *
```

데이터베이스 백업, 로그 정리 등에 가장 많이 사용되는 패턴입니다.

### 평일 오전 9시에 실행

```
0 9 * * 1-5
```

업무 시간 시작 시 리포트 생성이나 알림 발송에 활용합니다.

### 5분마다 실행

```
*/5 * * * *
```

헬스체크, 모니터링 데이터 수집에 적합합니다.

### 매월 1일 새벽 3시

```
0 3 1 * *
```

월간 정산, 리포트 생성에 사용합니다.

### 매주 일요일 새벽 2시

```
0 2 * * 0
```

주간 백업이나 시스템 점검 작업에 적합합니다.

## 자주 하는 실수

1. **시간대 혼동** — 서버의 시간대(UTC vs KST)를 반드시 확인하세요
2. **일(day)과 요일(weekday) 동시 지정** — 둘 다 지정하면 OR 조건으로 동작하여 예상보다 자주 실행됩니다
3. **`/` 시작값 오해** — `5/15`는 5분부터 15분 간격이지, 5분과 15분이 아닙니다

## 크론 표현식 검증하기

크론 표현식을 작성한 뒤에는 반드시 검증하세요. [크론 파서](/tools/net/cron-parser)에 표현식을 입력하면 다음 실행 시각과 사람이 읽을 수 있는 설명을 확인할 수 있습니다.

스케줄링에서 타임스탬프 변환이 필요하다면 [Unix 타임스탬프 변환기](/tools/net/unix-timestamp)도 함께 활용하면 편리합니다.

## 정리

크론 표현식은 5개 필드(분·시·일·월·요일)만 기억하면 됩니다. 처음에는 복잡해 보여도 몇 가지 패턴을 익히면 금방 익숙해집니다.
