## Base64 Is Not "Encryption"

Let's clear up the most common misconception first: **Base64 is encoding, not encryption.** Anyone can decode it and read the original. Base64's purpose isn't to hide data but to **carry binary data safely through channels that only allow text.**

When you need to send binary like images or files through text-based media — email, URLs, JSON, HTTP headers — Base64 steps in.

## How It Works — 3 Bytes Become 4 Characters

The core of Base64 is simple:

1. Re-slice the original data into **6-bit groups instead of 8-bit (1-byte) groups.**
2. Six bits range 0–63, so each maps to one of 64 safe characters (`A–Z`, `a–z`, `0–9`, `+`, `/`).
3. 3 bytes (24 bits) = 6 bits × 4 → so **3 bytes become 4 characters.**

That's why Base64 inflates data size by about **4/3 (≈ 33% larger)**. That's the cost of Base64.

Leftover slots are filled with `=` padding. If the input length isn't a multiple of 3, a `=` or `==` is appended.

```
input:  "Man" (3 bytes)  → "TWFu" (4 chars, no padding)
input:  "Ma"  (2 bytes)  → "TWE=" (1 padding)
input:  "M"   (1 byte)   → "TQ==" (2 padding)
```

## Where Base64 Is Used in Practice

- **data URI**: embedding a small image directly in HTML/CSS without a separate request — `data:image/png;base64,iVBOR...`. You can create these with the [Image to Base64 Converter](/tools/net/image-base64-converter/).
- **Email attachments (MIME)**: email is a text protocol, so attachments are Base64-encoded for transport.
- **JWT**: a token's Header and Payload are Base64URL-encoded (not encrypted — so no sensitive data).
- **HTTP Basic auth**: `Authorization: Basic <base64(user:pass)>`. It's only encoding, so it must be used with HTTPS.
- **Binary inside JSON**: passing small binary blobs as string fields in an API.

## Base64 vs Base64URL

Standard Base64 uses `+` and `/`, which have special meaning in URLs and filenames. So **Base64URL** swaps `+ → -`, `/ → _` and drops the `=` padding. It's used in JWTs, URL parameters, and the like. When using a converter, check which variant it is.

## Frequently Asked Questions (FAQ)

### Q. Can I store passwords as Base64?

Never. Base64 decodes trivially, so it offers zero security. Store passwords with a hash function like bcrypt.

### Q. Why does the file size grow?

Because 3 bytes are represented as 4 characters, it grows by about 33%. Inlining a large file as a data URI bloats the HTML and prevents caching, so reserve it for small icons.

### Q. Are data URIs always good?

For small assets they cut request counts and help, but for large images they hurt due to ① 33% size increase ② no browser caching ③ HTML bloat. Decide based on size and reuse frequency.

## Wrap-up

Base64 is "encoding that moves binary safely as text," not encryption. Remember just the principle — 3 bytes → 4 characters, ≈33% larger — and you won't get confused whether you meet it in a data URI, a JWT, or MIME. When you need to convert, use the [Base64 Encoder/Decoder](/tools/net/base64/).
