## 서브넷 마스크란?

서브넷 마스크(Subnet Mask)는 IP 주소에서 **네트워크 부분**과 **호스트 부분**을 구분하는 32비트 값입니다. 모든 TCP/IP 통신에서 라우팅 결정의 기초가 됩니다.

예를 들어, `255.255.255.0` (/24) 마스크는 IP 주소의 처음 24비트를 네트워크 식별에, 나머지 8비트를 호스트 식별에 사용합니다.

### 왜 서브네팅이 필요한가?

1. **IP 주소 절약**: 필요한 만큼만 호스트를 할당
2. **브로드캐스트 도메인 축소**: 네트워크 트래픽 감소
3. **보안 강화**: 네트워크 세그먼트 간 접근 제어

## CIDR 표기법

CIDR(Classless Inter-Domain Routing)은 기존 클래스 기반 주소 체계를 대체합니다. `/24`처럼 슬래시 뒤의 숫자가 네트워크 비트 수를 나타냅니다.

| CIDR | 서브넷 마스크 | 사용 가능 호스트 |
|------|-------------|----------------|
| /24  | 255.255.255.0 | 254 |
| /25  | 255.255.255.128 | 126 |
| /26  | 255.255.255.192 | 62 |
| /27  | 255.255.255.224 | 30 |
| /28  | 255.255.255.240 | 14 |

## VLSM 실무 활용

VLSM(Variable Length Subnet Masking)은 서브넷마다 다른 마스크 길이를 사용하여 IP 주소를 효율적으로 배분하는 기법입니다.

### 예제: 사무실 네트워크 설계

100명의 직원이 있는 회사에서 다음과 같이 부서별 네트워크를 나눌 수 있습니다:

- **개발팀** (50명): `192.168.1.0/26` → 62개 호스트
- **마케팅팀** (20명): `192.168.1.64/27` → 30개 호스트
- **경영진** (10명): `192.168.1.96/28` → 14개 호스트
- **서버 네트워크**: `192.168.1.112/28` → 14개 호스트

## 서브넷 계산 팁

서브넷 계산을 빠르게 하려면 **2의 거듭제곱**을 기억하세요:

```
2^1 = 2    2^5 = 32
2^2 = 4    2^6 = 64
2^3 = 8    2^7 = 128
2^4 = 16   2^8 = 256
```

호스트 수 = 2^(32 - 프리픽스 길이) - 2

> `-2`는 네트워크 주소와 브로드캐스트 주소를 제외하기 위함입니다.
