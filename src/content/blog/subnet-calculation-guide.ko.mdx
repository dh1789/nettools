## 서브넷 계산이 필요한 상황

서버 인프라를 구축하거나 클라우드 VPC를 설정할 때, "이 네트워크에 호스트가 몇 개 들어가지?"라는 질문을 마주하게 됩니다. 서브넷 계산은 네트워크 엔지니어뿐 아니라 DevOps, 클라우드 아키텍트에게도 필수 역량입니다.

## 서브넷 계산의 핵심 공식

서브넷 계산에서 가장 중요한 두 가지 공식입니다:

- **사용 가능 호스트 수** = 2^(32 - 프리픽스) - 2
- **서브넷 수** = 2^(빌린 비트 수)

`-2`는 네트워크 주소(첫 번째)와 브로드캐스트 주소(마지막)를 제외하기 위함입니다.

## 단계별 계산 방법

### 1단계: 필요한 호스트 수 파악

부서별 PC 수, 서버 수, 프린터 등 네트워크에 연결할 장비를 모두 세어봅니다. 향후 증가분(보통 20~30%)도 고려하세요.

### 2단계: 적절한 프리픽스 선택

| 필요 호스트 | 프리픽스 | 실제 호스트 수 |
|------------|---------|--------------|
| 1~2개      | /30     | 2            |
| 3~14개     | /28     | 14           |
| 15~30개    | /27     | 30           |
| 31~62개    | /26     | 62           |
| 63~126개   | /25     | 126          |
| 127~254개  | /24     | 254          |

### 3단계: 네트워크 범위 확인

`192.168.1.0/26`을 예로 들면:

- **네트워크 주소**: 192.168.1.0
- **첫 번째 호스트**: 192.168.1.1
- **마지막 호스트**: 192.168.1.62
- **브로드캐스트**: 192.168.1.63
- **다음 서브넷 시작**: 192.168.1.64

## 실전 예시: AWS VPC 서브넷 설계

`10.0.0.0/16` VPC에서 3개 가용영역(AZ)에 퍼블릭/프라이빗 서브넷을 배치하는 경우:

```
AZ-a 퍼블릭:  10.0.1.0/24  (254 호스트)
AZ-a 프라이빗: 10.0.11.0/24 (254 호스트)
AZ-b 퍼블릭:  10.0.2.0/24  (254 호스트)
AZ-b 프라이빗: 10.0.12.0/24 (254 호스트)
AZ-c 퍼블릭:  10.0.3.0/24  (254 호스트)
AZ-c 프라이빗: 10.0.13.0/24 (254 호스트)
```

> AWS는 각 서브넷에서 5개 IP를 예약하므로 실제 사용 가능 호스트는 251개입니다.

## 자주 하는 실수

1. **네트워크/브로드캐스트 주소에 호스트 할당** — 충돌 발생
2. **서브넷 겹침** — VPC 피어링이나 VPN 연결 시 라우팅 문제 유발
3. **여유 없는 설계** — 나중에 확장이 불가능

## 도구로 빠르게 검증하기
